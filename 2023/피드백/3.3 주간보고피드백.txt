3.3. 교수님 코멘트

1차심사 가시권... 팀워크 다져서 업무분배 잘할 것

졸업작품 = Engineering Sample
상품 개념 있으나 상품화 단계 X
적용되는 원리 검증 단계
ex: 라커보기가 적합한지? 스태빌라이저 성능 충분한지?

→ "공학적 문제" 정의, Solution 탐색 ... 이를 위한 시험용 설계

실제 현장에서의 필요성, 경제성은 추후에라도 고려해 변동 가능,
지금 단계에선 공학적 문제 위주의 접근이 중요

1차심사도 공학적 문제 도출을 중요하게 볼 것


문제는 다각적, 다양... 전부 다룰 순 없음
"해결하고자 하는 공학적 문제가 무엇인가"



지금까지= 스터디: 문헌조사, Reading, 시장조사...
이제부터= Research: 공학적 개선점 직접 연구
ex) 노면 성질 - 모든 노면 대응 가능?


문제 도출은 Feasibility Study (Test) 통해서
ex) 최대장애물 규격? 이에 따른 바퀴크기? 극복 불가능하면 우회? 극복 가부 판단은?


Track → 사막 진흙 눈길 등 Wheel로 주행 불가능한 특수 노면에서 유리
→ 무겁고 느리다는 단점...


<이번주 할 일>
지금까지 도출한 걸 가지고 1차심사 대비

* 풀고자 하는 목적 설정
* 공학적 문제 도출
* 수반되는 기능, 구조 탐구
* 제한조건 도출, Solution 가정
* 실제 작동성, 설계, 성능 등도 부속해서 설정

공학적 문제 예시)
Rocker-Bogie의 무게중심을 낮추면서 경사적응 잘하는 System 개발


좋은 문제인가?
- 해결 가치도 없는 문제 (X)
- 해결 불가능한 문제 (X)
- 대학 레벨에서 해결 가능하고 가치가 있는 문제 (O)

접근 가능성, 해결 가능성 Feasibility 검토 필수

주된 공학적 문제 해결을 위주로 진행하되,
나머지도 가시적인 문제들은 가급적 개선하는 방향으로



공학적 문제와 별개로 "활용적 특성"도 졸업작품에서 유용한 지점

Engineering Sample → 제품 규격 제시 가능
ex) 도심주행에서 예상 장애물, 요구되는 바퀴 크기, 적합한 화물 용량, 필요한 등판 능력 등...
